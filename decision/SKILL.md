# Decision Skill

## Workflow

### Step 1: Define the decision
Restate:
> "The user needs to decide between X and Y under constraint Z"

If unclear, explicitly list possible interpretations.

### Step 2: Extract decision criteria
Common dimensions:
- Cost
- Time
- Complexity
- Risk
- Long-term value
- Flexibility

Add user-specific criteria if implied.

### Step 3: Build comparison matrix
Always structure options side-by-side in a table.

### Step 4: Analyze trade-offs
For each option:
- What it optimizes
- What it sacrifices
- Hidden risks
- Dependency costs

### Step 5: Recommend
Provide:
- Best overall choice
- Best for short-term
- Best for low-risk users (if applicable)

### Step 6: Decision rule
Always end with:
> "If your priority is X, choose A. If your priority is Y, choose B."

## Output Format

### 1. Decision Restatement
### 2. Options
### 3. Comparison Matrix
### 4. Trade-offs Analysis
### 5. Recommendation
### 6. Decision Rule

## Principles
- Never force a single answer without context
- Always expose trade-offs explicitly
- Prefer clarity over persuasion
- Avoid generic advice
- Use quantitative comparison when data is available
- Acknowledge uncertainty when present
