# Insight Synthesizer Skill

## Core Principle
McKinsey partner-level thinking: identify the one variable that changes everything,
remove 80% noise, keep only the factors that actually determine outcomes.

## Workflow

### Step 1: Inventory all inputs
List every factor, variable, opinion, and data point the user provided.
Do not filter yet.

### Step 2: Classify signal vs noise
For each factor, ask:
- Does this change the outcome? (Yes/No)
- Can the user control it? (Yes/No)
- Is it measurable? (Yes/No)

Discard factors that score: No on outcome change OR (No on control AND No on measurable).

### Step 3: Find the keystone variable
Among remaining factors, identify the ONE that:
- Has the highest leverage (small change -> large impact)
- Is most actionable by the user
- Is most time-sensitive

### Step 4: Compress to 3 lines
State:
1. The keystone variable
2. Why it dominates
3. What to do about it right now

### Step 5: Optional - residual risk
List 1-2 factors that could invalidate the insight, with probability estimate.

## Output Format

### 1. Full Factor Inventory
### 2. Signal vs Noise Classification
### 3. Keystone Variable
### 4. 3-Line Insight
### 5. Residual Risk (optional)

## Principles
- Always reduce, never expand
- Prefer one sharp insight over ten fuzzy ones
- Quantify when possible ("70% of the outcome depends on X")
- If you cannot find a keystone, say so honestly
- Never pad with filler insights
- Think in leverage ratios, not lists
